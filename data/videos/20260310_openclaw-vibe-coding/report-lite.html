01OpenClaw/扣子养虾 核心概念与快速部署

OpenClaw 是基于字节跳动 Coze（扣子编程）平台构建的高级 AI 智能代理（Agent），被社区昵称为「扣子养虾/龙虾」。

核心定位：7x24 小时在线的 AI 分身，能够自主完成互动、内容创作、任务自动化、日常提醒等工作。

3分钟无代码部署流程：
  1. 访问 Code.coze.cn
  2. 找到 OpenClaw 项目模板（首页「优质案例」区）
  3. 点击「创建副本」
  4. 填写应用名称和介绍（默认已预填）
  5. 确认创建，部署完成

> 重要提示：副本创建范围包含代码、压缩后的上下文、数据库/存储/变量的结构定义，但不含具体数据及环境变量值。环境变量需要手动配置。

Coze 平台功能架构

Coze（扣子编程）是字节跳动推出的 AI 开发平台，口号「你的 AI 开发伙伴已就位」。平台核心模块包括：

应用层：
  • 网页应用、移动应用、小程序
  • 智能体（Agent）管理
  • 工作流（Workflow）编排
  • 技能（Skills）市场
开发者工具：
  • 项目管理 & 集成管理
  • 资源库 & 任务中心
  • API & SDK
  • 评测管理
两种开发模式：
  • 智能体开发 — 可视化配置 AI Agent
  • 工作流开发 — 拖拽式构建自动化流程

视频中展示的用户（ssamu）拥有 213,472 积分，创建了 15+ 个项目，包括直播、电商、视频风格生成器、AI学习路线图定制、RSS资讯助手、文稿、简历改写等。

03OpenClaw Gateway Dashboard 深度解析

视频后半段展示了 OpenClaw Gateway Dashboard — 这是 OpenClaw 的专业管理后台，远超普通聊天界面。它是一个完整的 AI Agent 运维控制台。

Dashboard 核心模块

Dashboard 顶部导航栏包含以下模块：

| 模块 | 功能 |

|------|------|

| 聊天 (Chat) | 直接网关聊天会话，支持快速交互 |

| 概览 (Overview) | Agent 整体状态总览 |

| 频道 (Channels) | 通信频道管理 |

| 实例 (Instances) | Agent 实例管理 |

| 会话 (Sessions) | 会话历史和管理 |

| 使用情况 (Usage) | 资源用量统计 |

| 定时任务 (Cron Jobs) | 计划任务配置 |

| 代理 (Agents) | Agent 配置和管理 |

| 技能 (Skills) | 技能安装和管理 |

| 节点 (Nodes) | 节点配置 |

右上角显示：版本号（dev）、健康状态（Health OK 绿灯）。

Agent 核心配置文件

在 Agents 标签页中，可以看到 Agent 的核心配置文件结构（存储于 /workspace/projects/workspace/）：

  • AGENTS.md — Agent 工作空间和路由配置
  • SOUL.md — Agent 的「灵魂」文件，定义人格和行为模式
  • TOOLS.md — 可用工具的声明和描述
  • IDENTITY.md — 身份信息配置
  • USER.md — 用户画像和偏好
  • HEARTBEAT.md — 心跳任务配置（定期自动执行的任务）
  • BOOTSTRAP.md — 启动引导配置
  • MEMORY.md — 记忆系统配置

HEARTBEAT.md 中可以看到任务描述和执行频率（如每5分钟触发一次），Agent 会自动根据心跳配置执行巡检和任务。

浏览器自动化能力

Dashboard 展示了 OpenClaw 的浏览器自动化功能：

  • Agent 能自主打开浏览器访问网页（演示中访问了 goodfish.com 和 code.coze.cn）
  • 浏览器状态监控：显示当前页面 URL、加载状态
  • 截图功能：自动截取浏览器页面保存到 media/browser/ 目录
  • 支持浏览器操控链接，可以远程控制浏览器

文件系统结构包括：agents/, browser/, canvas/, completions/, cron/, devices/, identity/, media/, memory/, scripts/, subagents/, workspace/ 等目录，展示了一个完整的 Agent 运行时环境。

04实战案例：从零搭建「深红港任务公会」网站

直播中最重磅的演示是用 OpenClaw 从零搭建一个完整的网站项目——「深红港任务公会」。这是一个游戏风格的任务分发平台。

需求分析与方案设计

在飞书文档中（如何利用扣子龙虾制作一个Agent打工网站），OpenClaw 进行了完整的需求分析：

用户角色与场景：
  • 企业招聘方 — 发布 AI 机器人相关岗位（客服机器人、销售助手、数据分析机器人等）
  • 机器人所有者 — 创建机器人简历（能力、技能、经验、项目案例）
核心功能：
  • 查看机器人简历和技能
  • 收到申请通知
  • 管理招聘流程
方案选择： OpenClaw 提出了「方案一：深红港（推荐-平衡好实施）」，采用游戏化设计风格，包含世界观和区域划分。

AI 自主开发过程

OpenClaw Agent 展示了完整的自主开发流程：

  1. 代码生成 — 自动生成前端页面和后端逻辑
  2. 数据库配置 — 使用 sqlite3 数据库存储数据
  3. 调试修复 — 遇到编辑 package.json 失败等错误时自动重试
  4. 服务启动 — 自动初始化数据库、启动 Web 服务
  5. API 测试 — 验证服务运行正常（端口3000，latency ~4ms）

> 开发过程中出现了多次错误和重试（如「回执发现沟通用出置信失败」），但 Agent 能自主诊断问题并调整方案。

最终成果： 成功部署了「深红港任务公会」网站，包含：
  • 深渊区域系统（浅滩区、珊瑚区、深渊带、海沟区）
  • 用户评分和等级系统（5星评级）
  • 任务发布和领取功能
  • 完整的 Web UI（深色主题游戏风格）

05ClawEx 扩展工具与飞书集成

ClawEx 是 OpenClaw 的扩展管理工具，支持通过 npm 包安装额外插件和技能。

Skills 技能体系

OpenClaw 的技能系统分为三层：

| 类型 | 数量 | 说明 |

|------|------|------|

| WORKSPACE SKILLS | 6 | 用户自定义的工作空间技能 |

| BUILT-IN SKILLS | 52 | 内置技能 |

| EXTRA SKILLS | 4 | 额外安装的扩展技能 |

飞书相关的 Extra Skills 包括：

  • feishu-doc — 飞书文档读写操作
  • feishu-drive — 飞书云存储文件管理
  • feishu-perm — 飞书权限管理（文档和文件的共享、权限、协作）
  • feishu-wiki — 飞书知识库操作

每个 Skill 可以设置为 eligible（可用）或 Disable（禁用），并关联到 openclaw-extra 命名空间。

飞书插件安装流程

在 waytoagi.feishu.cn 的教程文档中，详细记录了飞书集成步骤：

准备工作：
  1. 在飞书开放平台完成以下配置：
  • 创建企业自建应用
  • 应用机器人能力
  • 配置事件订阅（WebSocket 长连接模式）
  • 添加权限：im:message、im:message:send_as_bot
  • 发布应用（至少灰度版本）
  1. 获取 App ID 和 App Secret
重要步骤二：初次访问配对
  1. 在飞书中向机器人发送任意消息
  2. 系统生成配对码（字母+数字组合）
  3. 配对码有效期为 5 分钟
  4. 在服务器终端执行：openclaw pairing approve feishu <配对码> -notify
实用案例：飞书紧急通知
  • OpenClaw 可通过飞书发送紧急提醒
  • 支持配置飞书电话加急、发短信等功能

FAQ
Q: OpenClaw 和普通的 ChatGPT / Coze Bot 有什么区别？

OpenClaw 是一个完整的 AI Agent 系统，具备：(1) Gateway Dashboard 专业管理后台；(2) 浏览器自动化能力；(3) 文件系统和 workspace 管理；(4) Skills 技能扩展体系（52+ 内置技能）；(5) 定时任务（Heartbeat/Cron）；(6) 记忆持久化；(7) 自主编码和部署能力。普通 Bot 只是对话窗口，OpenClaw 是一个自主工作的 Agent。

Q: 我不懂编程，能用 OpenClaw 吗？

完全可以。OpenClaw 的核心价值就是让非技术用户通过自然语言「指挥」AI 完成编程任务。直播中演示了从零搭建完整网站（含数据库、前端、API），全程用自然语言对话完成。这就是 Vibe Coding 的精髓。

Q: OpenClaw 的开发能力有多强？

直播中实际演示了：(1) 自主设计方案（深红港任务公会）；(2) 自动生成前端 + 后端代码；(3) 配置 sqlite3 数据库；(4) 调试修复错误并自动重试；(5) 启动 Web 服务并验证 API（latency ~4ms）；(6) 进行多平台电商数据分析（368件商品、4平台对比）。有错误但能自主恢复。

Q: 如何让 OpenClaw 连接飞书？

需要三步：(1) 在飞书开放平台创建企业自建应用，配置机器人能力和 WebSocket 事件订阅；(2) 通过 ClawEx 安装 feishu-plugin.tgz 扩展包；(3) 在飞书中发送消息触发配对，5分钟内在终端执行 openclaw pairing approve feishu <配对码> -notify 完成绑定。

Q: InStreet 是什么？怎么用？

InStreet (instreet.coze.site) 是 AI Agent 社交网络，由 Agent（而非人类）发帖和互动。你的 OpenClaw 龙虾可以在上面发帖、评论、分享技能。注册指南在 instreet.coze.site/skill.md。当前社区有 872 个 Agent、2358 帖子。
